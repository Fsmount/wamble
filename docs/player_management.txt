Wamble Player Management
=======================

Overview
- Keeps an in-memory table of active players and their session tokens.
- Supports ephemeral sessions and sessions linked to a persistent public key.
- Reads and writes the bits that belong in the database.

Tokens
- 16 raw bytes per session. Logged as a 22-char base64url string.
- Generated with a simple PCG RNG and extra OS entropy where available.
- We check for collisions in memory, and the persistence layer treats tokens as `UNIQUE`.

Creating Players
- Ephemeral: `create_new_player()` mints a token, creates a DB session, sets defaults (rating, score, timers).
- Persistent attach: `attach_persistent_identity(token, pubkey)` links an existing
  anonymous session (created by `CLIENT_HELLO`) to a pubkey-backed identity
  without changing the session token.
- Login authentication uses a challenge-response flow:
  - client sends `LOGIN_REQUEST` with `public_key[32]`.
  - server responds with `LOGIN_CHALLENGE` (`challenge[32]`).
  - client signs protocol context + token + public key + challenge using Ed25519
    and sends `LOGIN_REQUEST` with `public_key[32] + signature[64]`.
  - server verifies the signature before linking identity.

Lookups
- `get_player_by_token(token)` returns a known player. If only the DB knows about the token, we rehydrate a minimal player (score, rating, games, chess960 games) and update last-seen.

Stats
- Player stats track both:
  - total games played.
  - games played on Chess960 boards.
- `GET_PLAYER_STATS` / `PLAYER_STATS_DATA` includes the Chess960 game counter in
  the payload.
- `GET_PLAYER_STATS` may target another player via extension fields:
  - `stats.request_id` (`int`) for request/response correlation.
  - `stats.target_session_id` (`int`).
  - `stats.target_handle` (`string`, global opaque handle).
- `PLAYER_STATS_DATA` echoes target metadata via extensions:
  `stats.request_id`, `stats.target_session_id`, `stats.target_has_identity`,
  `stats.target_handle`.

Expiration
- `player_manager_tick()` drops idle entries after `token-expiration` seconds and removes the token from the index.

Config
- `max-players`, `token-expiration`, `default-rating`, `max-token-attempts`, `max-token-local-attempts`.

API
- `player_manager_init()`, `get_player_by_token()`, `create_new_player()`,
  `attach_persistent_identity()`, `player_manager_tick()`.
