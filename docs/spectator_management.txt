Wamble Spectator Management
==========================

Overview
- Lets a session watch games in two modes: a rotating summary or a single focused board.
- Uses `SPECTATE_UPDATE` packets over UDP for both reliable state snapshots and
  usually unreliable live refreshes.

States
- `IDLE`: not watching.
- `SUMMARY`: periodic batches of interesting boards.
- `FOCUS`: periodic updates about one board.

Config
- `max-spectators`, `spectator-visibility`, `spectator-summary-hz`,
  `spectator-focus-hz`, `spectator-max-focus-per-session`,
  `spectator-summary-mode`.
- Capacity bypass for focused mode is policy-controlled via
  `spectate.capacity_bypass` / `focus`, and bypass-focused sessions are counted
  outside the non-bypass capacity pool.

Requests
- Summary: send `SPECTATE_GAME` with `board_id=0` -> `SPECTATOR_OK_SUMMARY`.
  - Optional mode filter flags on summary requests:
    - `WAMBLE_FLAG_MODE_FILTER_CHESS960` to include Chess960 boards.
    - `WAMBLE_FLAG_MODE_FILTER_STANDARD` to include standard boards.
    - Omitting both flags includes all eligible boards.
  - If `game.mode` / `view` is not allowed for the session, mode filter flags
    are ignored.
  - Terminal response: reliable summary snapshot made of one or more
    `SPECTATE_UPDATE` packets with `spectate.state=summary`, including a
    reset packet (`board_id=0`, empty `fen`) carrying
    `spectate.summary_generation`.
- Focus: send `SPECTATE_GAME` with `board_id>0` -> `SPECTATOR_OK_FOCUS` if eligible.
  - Terminal response: reliable `SPECTATE_UPDATE` snapshot with
    `spectate.state=focus`.
- Stop: `SPECTATE_STOP` -> `SPECTATOR_OK_STOP`.
  - Terminal response: reliable `SPECTATE_UPDATE` snapshot with
    `spectate.state=idle`.
- Low trust or capacity issues return an error (see enum values).
- For reliable spectate requests, the terminal `SPECTATE_UPDATE` or denial
  `ERROR` is sent before the request-level `ACK`.

Updates
- Collected by `spectator_collect_updates()` at the configured rate per mode.
- Ordering favors recent boards, then a simple attractiveness score.
- `SPECTATE_UPDATE` sets `WAMBLE_FLAG_BOARD_IS_960` only when:
  - the updated board is Chess960, and
  - the spectator session is allowed to view game mode (`game.mode` / `view`).
- If a focused game ends or focus gets disabled, we queue an explicit notice
  (type `SPECTATE_ENDED`) and switch back to summary.
- If `max-spectators` is reduced to `0`, non-bypass spectators are explicitly
  stopped (type `SPECTATE_ENDED`) rather than silently left active.
- State-changing notices are paired with a reliable `SPECTATE_UPDATE` sync so
  the client has one authoritative post-transition state packet.

Notifications
- `spectator_collect_notifications()` returns pending notices as
  `SpectatorUpdate` structs including a `notification_type` field.
- Notifications are sent as `SERVER_NOTIFICATION` packets with a typed
  1-byte prefix before the text body.
- Types used by the spectator system:
  - `WAMBLE_NOTIFICATION_TYPE_SPECTATE_ENDED` (`2`): focus ended or
    spectating stopped.

Port Affinity
- Entries are tied to the listener port that created them; only that thread emits updates for that entry.

API
- `spectator_manager_init()`, `spectator_manager_shutdown()`, `spectator_manager_tick()`.
- `spectator_handle_request()`, `spectator_collect_updates()`, `spectator_collect_notifications()`.
