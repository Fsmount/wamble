Wamble Database
================

Overview
- Backend: PostgreSQL.
- Purpose: Persist players/sessions, boards, moves, reservations, results, payouts, and predictions.
- Profiles: Each advertised profile uses its own per-thread database connection.
- Access: Runtime code reads via a small `WambleQueryService` interface and writes by enqueueing persistence intents which are flushed to the database in batches.

Connections
- Per-thread per-listener game DB: Each listener thread opens its own profile DB
  connection using `db-host`, `db-user`, `db-pass`, `db-name`.
- Shared global DB: Identity and policy resolution uses a separate shared store
  connection configured by `global-db-host`, `global-db-user`,
  `global-db-pass`, `global-db-name`.

Read Path Caching
- Token-indexed reads are cached in thread-local memory to reduce repeated DB lookups
  during hot loops (session lookup, persistent-session lookup).
- Cache characteristics:
  - Capacity: 256 entries per cache type per runtime thread.
  - TTL: 500ms per entry.
  - Scope: thread-local; no cross-thread sharing.
- Cache entries include both successful and failure statuses (`DB_OK`, `DB_NOT_FOUND`,
  connection/exec/bad-data errors), so repeated failures are also throttled within TTL.
- Session-linking updates invalidate the persistent-session cache for correctness.

Write Batching and Transactions
- Intent flushes are applied in bounded batches (intent count and estimated payload bytes;
  see `persistence-max-intents` and `persistence-max-payload-bytes` in config).
- The selected batch is attempted inside an explicit DB transaction (`BEGIN`/`COMMIT`).
- If transactional apply fails, the runtime rolls back and retries selected intents in
  non-transactional per-intent mode, requeuing only failed intents.
- Intent selection/coalescing rules reduce redundant writes before execution:
  - Keep only the newest per key for `UPDATE_BOARD`, `UPDATE_BOARD_ASSIGNMENT_TIME`,
    `UPDATE_SESSION_LAST_SEEN`, and `REMOVE_RESERVATION`.
  - `RECORD_MOVE` and `RECORD_PAYOUT` intents are grouped and sorted for append-friendly
    write locality.
- Flush cadence is runtime-driven and can perform multiple batches per cycle when backlog
  is high.

Tables
- players
  - `id BIGSERIAL PRIMARY KEY`
  - `public_key BYTEA UNIQUE NOT NULL` (32 bytes)
  - `rating DECIMAL(10,4) NOT NULL`
  - `created_at TIMESTAMPTZ DEFAULT NOW()`

- sessions (profile DB)
  - `id BIGSERIAL PRIMARY KEY`
  - `token BYTEA UNIQUE NOT NULL` (16 bytes)
  - `player_id BIGINT NULL REFERENCES players(id)`
  - `global_identity_id BIGINT NOT NULL` (identity id allocated in shared identity DB)
  - `config_revision_id BIGINT`
  - `policy_snapshot_revision_id BIGINT`
  - `total_score DECIMAL(10,4) NOT NULL DEFAULT 0`
  - `total_prediction_score DECIMAL(10,4) NOT NULL DEFAULT 0`
  - `games_played INTEGER NOT NULL DEFAULT 0`
  - `treatment_group_key VARCHAR(128)`
  - `treatment_rule_id BIGINT`
  - `treatment_snapshot_revision_id BIGINT`
  - `treatment_assigned_at TIMESTAMPTZ`
  - `created_at TIMESTAMPTZ DEFAULT NOW()`
  - `last_seen_at TIMESTAMPTZ DEFAULT NOW()`
  - Indexes: `idx_sessions_token` on `(token)`, `idx_sessions_identity_id` on `(global_identity_id)`, `idx_sessions_treatment_group` on `(treatment_group_key)`

- global_identities (shared global DB)
  - `id BIGSERIAL PRIMARY KEY`
  - `public_key BYTEA UNIQUE NULL` (32 bytes for persistent anonymous IDs)
  - `created_at TIMESTAMPTZ DEFAULT NOW()`

- global_policy_rules (shared global DB)
  - `id BIGSERIAL PRIMARY KEY`
  - `global_identity_id BIGINT NOT NULL`
  - `action VARCHAR(128) NOT NULL`
  - `resource VARCHAR(256) NOT NULL`
  - `scope VARCHAR(256) NOT NULL DEFAULT '*'`
  - `effect VARCHAR(8) NOT NULL` in {'allow','deny'}
  - `permission_level INTEGER NOT NULL DEFAULT 0`
  - `policy_version VARCHAR(64) NOT NULL DEFAULT 'v1'`
  - `source VARCHAR(32) NOT NULL DEFAULT 'manual'`
  - `snapshot_revision_id BIGINT`
  - `not_before_at TIMESTAMPTZ`
  - `not_after_at TIMESTAMPTZ`
  - `context_key VARCHAR(128)`
  - `context_value VARCHAR(256)`
  - `reason TEXT`
  - `created_at TIMESTAMPTZ DEFAULT NOW()`
  - `updated_at TIMESTAMPTZ DEFAULT NOW()`
  - Indexes: `(global_identity_id)`, `(action, resource)`,
    `(global_identity_id, action, scope, resource)`, `(source, snapshot_revision_id)`

- global_identity_tags (shared global DB)
  - `global_identity_id BIGINT NOT NULL`
  - `tag VARCHAR(128) NOT NULL`
  - `created_at TIMESTAMPTZ DEFAULT NOW()`
  - Primary key: `(global_identity_id, tag)`
  - Indexes: `(tag, global_identity_id)`

- global_treatment_groups (shared global DB)
  - `id BIGSERIAL PRIMARY KEY`
  - `group_key VARCHAR(128) NOT NULL`
  - `priority INTEGER NOT NULL`
  - `is_default BOOLEAN NOT NULL`
  - `source VARCHAR(32) NOT NULL`
  - `snapshot_revision_id BIGINT`

- global_treatment_assignment_rules (shared global DB)
  - `id BIGSERIAL PRIMARY KEY`
  - `global_identity_id BIGINT NOT NULL`
  - `profile_scope VARCHAR(256) NOT NULL`
  - `group_key VARCHAR(128) NOT NULL`
  - `priority INTEGER NOT NULL`
  - `source VARCHAR(32) NOT NULL`
  - `snapshot_revision_id BIGINT`

- global_treatment_assignment_predicates (shared global DB)
  - `rule_id BIGINT NOT NULL`
  - `fact_key VARCHAR(128) NOT NULL`
  - `op VARCHAR(16) NOT NULL`
  - typed predicate value columns

- global_treatment_group_edges (shared global DB)
  - `source_group_key VARCHAR(128) NOT NULL`
  - `target_group_key VARCHAR(128) NOT NULL`
  - `source VARCHAR(32) NOT NULL`
  - `snapshot_revision_id BIGINT`

- global_treatment_group_outputs (shared global DB)
  - `id BIGSERIAL PRIMARY KEY`
  - `group_key VARCHAR(128) NOT NULL`
  - `hook_name VARCHAR(64) NOT NULL`
  - `output_kind VARCHAR(32) NOT NULL`
  - `output_key VARCHAR(128) NOT NULL`
  - typed value columns / fact reference
  - `source VARCHAR(32) NOT NULL`
  - `snapshot_revision_id BIGINT`

- global_runtime_config_blobs (shared global DB)
  - `content_hash VARCHAR(32) PRIMARY KEY` (md5 of config text)
  - `config_text TEXT NOT NULL`
  - `created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`
  - Purpose: deduplicated config payload storage.

- global_runtime_config_revisions (shared global DB)
  - `id BIGSERIAL PRIMARY KEY`
  - `profile_key VARCHAR(128) NOT NULL`
  - `content_hash VARCHAR(32) NULL REFERENCES global_runtime_config_blobs(content_hash)`
  - `source VARCHAR(32) NOT NULL` (e.g. `file`, `runtime`)
  - `result VARCHAR(16) NOT NULL` (`active` or `rejected`)
  - `error_text TEXT`
  - `created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`
  - `activated_at TIMESTAMPTZ`
  - Indexes:
    - `(profile_key, created_at DESC)` for attempt-order lookup
    - `(profile_key, result, created_at DESC)` for active-entry lookup
  - Purpose: full history of attempts, including invalid/unloadable configs.

- boards
  - `id BIGSERIAL PRIMARY KEY`
  - `fen VARCHAR(90) NOT NULL`
  - `status VARCHAR(16) NOT NULL` in {'ACTIVE','RESERVED','DORMANT','ARCHIVED'}
  - `last_assignment_time TIMESTAMPTZ`
  - `last_mover_treatment_group VARCHAR(128)`
  - `created_at TIMESTAMPTZ DEFAULT NOW()`
  - `updated_at TIMESTAMPTZ DEFAULT NOW()`
  - Indexes: `idx_boards_status` on `(status)`
  - Trigger: `trigger_boards_updated_at` updates `updated_at` on update.

- moves
  - `id BIGSERIAL PRIMARY KEY`
  - `board_id BIGINT NOT NULL REFERENCES boards(id)`
  - `session_id BIGINT NOT NULL REFERENCES sessions(id)`
  - `move_uci VARCHAR(6) NOT NULL`
  - `move_number INTEGER NOT NULL`
  - `timestamp TIMESTAMPTZ DEFAULT NOW()`
  - Indexes: `(board_id)`, `(session_id)`

- reservations
  - `board_id BIGINT PRIMARY KEY REFERENCES boards(id)`
  - `session_id BIGINT NOT NULL REFERENCES sessions(id)`
  - `expires_at TIMESTAMPTZ NOT NULL`
  - Index: `idx_reservations_expires_at` on `(expires_at)`

- game_results
  - `board_id BIGINT PRIMARY KEY REFERENCES boards(id)`
  - `winning_side CHAR(1) NOT NULL` in {'w','b','d'}
  - `finished_at TIMESTAMPTZ DEFAULT NOW()`

- payouts
  - `id BIGSERIAL PRIMARY KEY`
  - `board_id BIGINT NOT NULL REFERENCES boards(id)`
  - `session_id BIGINT NOT NULL REFERENCES sessions(id)`
  - `points_awarded DECIMAL(10,4) NOT NULL`
  - `created_at TIMESTAMPTZ DEFAULT NOW()`
  - Index: `idx_payouts_session_id` on `(session_id)`
  - Trigger: `trg_payouts_bump_session_total_score` increments
    `sessions.total_score` on insert.

- predictions
  - `id BIGSERIAL PRIMARY KEY`
  - `board_id BIGINT NOT NULL REFERENCES boards(id)`
  - `session_id BIGINT NOT NULL REFERENCES sessions(id)`
  - `predicted_move_uci VARCHAR(6) NOT NULL`
  - `move_number INTEGER NOT NULL DEFAULT 0`
  - `parent_prediction_id BIGINT REFERENCES predictions(id)`
  - `correct_streak INTEGER NOT NULL DEFAULT 0`
  - `points_awarded DECIMAL(10,4) NOT NULL DEFAULT 0`
  - `resolved_at TIMESTAMPTZ`
  - `status VARCHAR(16) NOT NULL DEFAULT 'PENDING'` in {'PENDING','CORRECT','INCORRECT','EXPIRED'}
  - `created_at TIMESTAMPTZ DEFAULT NOW()`
  - Indexes: `idx_predictions_board_status_move` on `(board_id, status, move_number)`, `idx_predictions_session_status` on `(session_id, status, created_at DESC)`

  `global_policy_rules` in the global DB.

  snapshots used for reload restore.
- 007_global_policy_runtime_expansion.sql: expands global policy rule shape
  (`scope`, `source`, snapshot binding, time/context predicates).
- 008_profile_identity_sessions.sql: profile DB migration for session identity
  linkage (`sessions.global_identity_id`).
- 009_global_identity_tags.sql: adds identity tags for bulk policy targeting.
- 010_profile_prediction_resolution.sql: adds prediction resolution columns and
  per-session prediction totals.

  assignment rules, predicates, outputs, and directed edges.
- 011_profile_treatment_groups.sql: adds per-session treatment assignment
  columns and board last-mover treatment metadata.

Trust and Visibility
- Client trust resolves from identity-scoped policy only, in deterministic
  precedence:
  1) resource `profile:<profile_name>`
  2) resource `profile_group:<profile_group>`
  3) resource `*`
- Action used by current trust-tier bridge is `trust.tier`.
- Deny precedence is explicit at each scope level (`effect='deny'` beats allow
  for equal scope precedence).
- Session trust is always policy-derived.
- LIST_PROFILES filtering compares profile visibility against the resolved
  trust tier.
- GET_PROFILE_INFO applies the same discovery authorization as LIST_PROFILES.

Operational Notes
- Multiple advertised profiles within one process may point to different databases.
- Persistence intents are applied from the runtime loop (threaded for multi-profile, inline
  in main thread for single-runtime mode).
- On startup/reload, the server records config attempts into
  `global_runtime_config_revisions`.
- Config restore queries the newest row where `result='active'`, while
  operators can query attempt history independently.

Examples
- Set trust level 10 globally for a known identity id:
  INSERT INTO global_policy_rules (global_identity_id, action, resource, scope, effect, permission_level, reason)
  VALUES (42, 'trust.tier', 'tier', '*', 'allow', 10, 'manual grant');

- Tag identities for bulk policy updates:
  INSERT INTO global_identity_tags (global_identity_id, tag)
  VALUES (42, 'ops'), (43, 'ops');

- Set trust level 6 for all identities with tag `ops` on profile group `staff`:
  INSERT INTO global_policy_rules (global_identity_id, action, resource, scope, effect, permission_level, reason)
  SELECT t.global_identity_id, 'trust.tier', 'tier', 'profile_group:staff', 'allow', 6, 'ops group grant'
  FROM global_identity_tags t
  WHERE t.tag = 'ops';

- Set trust level 10 globally for a persistent identity by public key:
  INSERT INTO global_policy_rules (global_identity_id, action, resource, effect, permission_level, reason)
  SELECT i.id, 'trust.tier', '*', 'allow', 10, 'manual grant'
  FROM global_identities i
  WHERE i.public_key = decode('<pubkey_hex>', 'hex')
  ;
